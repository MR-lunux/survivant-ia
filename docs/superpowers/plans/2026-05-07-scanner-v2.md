export interface Job {
  slug: string
  label: string

La heuristique baseline est correcte pour ~75% des jobs. Les ~50 jobs ci-dessous nécessitent une correction éditoriale.

**Logique des overrides** :
- **danger v1 → mutes v2** : jobs cognitifs où l'expertise se reconcentre sur l'audit/validation IA (comptable, analyste, journaliste...). Le risque reste élevé mais le potential aussi.
- **danger v1 → pivotes v2** : jobs purement exécutants où l'IA substitue, peu de levier d'amplification (saisie, télémarketing, transcription).
- **mutation v1 → garde mutes** : déjà correct dans 90% des cas. Ajustements de potential pour refléter la nuance.
- **protege v1 → pilotes v2** : jobs intellectuels protégés où l'IA augmente massivement (médecins, avocats, enseignants supérieurs).
- **protege v1 → garde tiens** : jobs manuels/physiques où l'IA n'amplifie pas (artisans, kinés, mécaniciens).
- **croissance v1 → tiens v2** : jobs en croissance demographique mais peu de levier IA (auxiliaire vie, aide-soignant, technicien manuel).
- **croissance v1 → garde pilotes** : jobs en croissance ET fort levier IA (ingénieur IA, expert cybersécurité, formateur).

  if (!selectedJob.value) return
  submitting.value = true
  submitError.value = false
  submitMessage.value = ''

  try {
    const res = await $fetch<{ ok: boolean }>('/api/subscribe', {

        job_quadrant: selectedJob.value.quadrant,
        job_risk: selectedJob.value.risk,
        job_potential: selectedJob.value.potential,
        formations_interest: formationsInterest.value,
      },
    })
    if (res.ok) {
      submitMessage.value = 'C\'est bon. Premier article dans 7 jours max.'
      emailInput.value = ''
      formationsInterest.value = false

    submitting.value = false
  }
}
